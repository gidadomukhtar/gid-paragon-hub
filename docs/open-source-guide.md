# 🌍 Open Source Guide

## What is Open Source?

Open source software is software whose source code is publicly available for anyone to view, use, modify, and distribute.

## Why contribute?

- Build real-world collaboration experience
- Improve coding and communication skills
- Grow a public portfolio recruiters can review
- Give back to the community

## How to make your first contribution

1. Find a beginner-friendly repository
2. Read `README` and `CONTRIBUTING` files
3. Choose an issue labeled `good first issue`
4. Ask clarifying questions in issue comments
5. Fork repo, create branch, implement changes
6. Open pull request with clear description

## Beginner-friendly repositories

- https://github.com/firstcontributions/first-contributions
- https://github.com/public-apis/public-apis
- https://github.com/EddieHubCommunity
- https://github.com/microsoft

## How to write a good pull request

- Explain what changed and why
- Reference issue number if applicable
- Include screenshots for UI changes
- Keep scope small and focused
- Request feedback politely

## Basic Git workflow

```bash
# fork + clone
git clone <your-fork-url>
cd <repo>

# create branch
git checkout -b feature/my-update

# make changes, then commit
git add .
git commit -m "docs: improve beginner guide"

# push branch
git push origin feature/my-update
```
